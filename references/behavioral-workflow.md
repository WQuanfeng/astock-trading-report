# 群体交易行为工作流

将“人性分析”限定为可观测的群体交易行为和供需结果。不得用价格、成交量反推
无法验证的交易者身份、心理活动或操纵意图。

## 分析边界

1. 使用“追涨意愿”“风险规避”“分歧与换手”“事件后的价格接受度”和“潜在
   供给压力”等可观察概念。
2. 不得写“主力控盘”“庄家出货”“散户恐慌”“机构吸筹”等身份或意图叙事，
   除非正式来源直接披露相应主体及行为。
3. “新资金”“持有人”“套牢盘”均不得作为已证实事实。没有逐账户或可靠筹码
   分布数据时，改写为“买盘接受度”“潜在卖压”或“前期密集成交区”。
4. 行为分析不得改变候选硬门槛、入选集合、`selection_rank` 或固定排序；只在
   候选确定后解释交易环境和次日节奏。
5. 只有证据充分度为“高”或“中”的行为结论，才可收紧次日触发或不介入条件；
   新条件仍须写成原子条件。行为结论可以支持既有计划，但不得删除或放宽既有
   条件。证据充分度为“低”或“不足”时不得调整计划。
6. 不得把行为结论直接写成买入或卖出指令。

## 按数据能力收集

- 市场层：上涨/下跌和跌停扩散、涨停/炸板/回封生态、连板结构、昨日涨停次日
  表现、开盘缺口、成交额、波动及其 T-1 和近期基准。
- 个股层：开高低收、换手率、成交量相对近 20 日基准、相对板块表现，以及可得
  时带时间戳的 5 分钟或更细价量序列。
- 事件层：公告或新闻的原始发布时间、价格开始反应的时间、同题材或板块同期
  表现；事件发布时间晚于价格反应时不得建立因果解释。
- 供给压力：只有数据源明确提供成交价分布、筹码分布，或存在可重复验证的放量
  受阻证据时才分析。均线或单个前高本身不能证明套牢盘。
- 资金流、龙虎榜和单一席位信息只作辅助，不得单独证明群体心理或主体意图。

数据源没有返回盘口、回封时间、分钟 K 线或成交价分布时，直接记录数据缺口；
不得用日线形态补写盘中先后顺序。

## 选择关键行为维度

JSON 的 `behavior_type` 只使用下列稳定枚举，英文枚举不得显示在 HTML 正文中：

| 中文维度 | `behavior_type` | 主要可验证代理 |
| --- | --- | --- |
| 追涨意愿 | `chasing` | 连板结构、昨日涨停溢价、炸板/回封、强势股高开后的价格接受度 |
| 风险规避 | `risk_aversion` | 涨跌广度、跌停与破位扩散、低开修复、波动和成交放大 |
| 分歧与换手 | `disagreement` | 换手相对基准、冲高回落或回封时序、日内量价和相对强弱 |
| 事件价格接受度 | `event_acceptance` | 事件时间后的价格路径、量能、板块和同题材相对表现 |
| 潜在供给压力 | `supply_pressure` | 成交价分布、密集成交区反应、反弹持续性和放量受阻证据 |

市场只保留最多三条对次日计划最重要的行为结论；每只请求股票或最终候选只保留
最多两条。没有重要且可验证的结论时输出“数据不足”，不得凑数。

## 固定分析格式

每条结论依次写入并在 HTML 中按相同顺序展示：

1. **事实**：只写带日期、证据 ID，以及在判断先后顺序时所需时间戳的观察值。
2. **计算**：写明指标、公式、结果、单位、比较基准和基准窗口。
3. **有限解读**：使用“显示……增强/减弱”“更符合……”等可撤销措辞。
4. **反证**：至少一项与当前解读冲突的事实；确无可用反证时写明缺口。
5. **次日观察**：给出可衡量条件和时间窗口，并写入原子条件对象。

同时列出数据缺口和证据充分度。证据充分度仅描述数据覆盖与证据一致性，不是
上涨概率或预测置信率：

- **高**：所需数据粒度和时序完整，至少两项不重复的基准化计算一致，且核心
  反证已处理。
- **中**：核心时序可用，至少一项基准化计算和另一项事实相互支持，非核心数据
  可以缺失。
- **低**：只有日线汇总、覆盖不完整或存在未解决的重要反证。
- **不足**：缺少形成该结论的核心数据，只能陈述已知事实。

## 对次日计划的影响

HTML 使用下列中文，JSON 使用对应的 `plan_effect`：

| HTML 显示 | `plan_effect` |
| --- | --- |
| 支持等待触发后选择性介入 | `supports_selective_entry` |
| 等待进一步确认 | `wait_for_confirmation` |
| 不适合新开仓 | `avoid_new_entry` |
| 不改变原计划 | `no_plan_change` |
| 数据不足，不调整计划 | `insufficient_data` |

`supports_selective_entry` 仍不等于买入；个股必须满足既有介入条件。
`avoid_new_entry` 也不等于自动卖出；已有持仓只能依据用户持仓信息和既有失效/
退出条件另行判断。

## T-1 复盘

将次日观察同时写入 `next_session_observations` 和
`observation_condition_rules`。下一交易日仅评价已记录的原子条件，不得根据
结果重写昨日解读。行为项目可写入 `previous_plan_verification`，但省略
`selection_outcome`，不得把行为确认率混入候选方向正确率。
