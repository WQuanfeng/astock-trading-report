# 个股每日工作流

## 使用 a-stock-data 收集

对用户请求的股票和选定已完成交易日，收集：

- 最新报价和日 K 线历史，并明确复权口径。
- 使用 `a-stock-data` 已提供的数据/指标获得的成交量、换手率、相对位置、
  均线状态和近期价格表现。
- 行业/概念归属及板块当日强弱。
- 当日及近期公告、公司回复和新闻；保留原始来源、时间及是否为正式披露。
- 可得资金流、龙虎榜、重大解禁/减持事件和重要风险信息。
- 可得时收集 5 分钟或更细的价量序列、开盘缺口、日内相对板块表现，以及数据源
  明确提供的成交价分布；不得自行把均线或单个前高解释为套牢盘。
- 停复牌、特别处理、适用涨跌停锁定状态、流动性、已知次日限制和临近公司行动。
- 按 `decision-integrity-rules.md` 预先确定的同期评价基准及其稳定代码。
- 该股票已存在时对应的 T-1 context。

## 按以下顺序解读

1. 区分正式事实与媒体解读。
2. 在板块与市场状态背景下解释量价行为。
3. 识别支持趋势延续和反对趋势延续的证据。
4. 仅复盘昨日记录的原子触发条件或失效条件是否发生。
5. 按 `behavioral-workflow.md` 分析买盘接受度、分歧和潜在供给压力；不得识别
   无法验证的交易者身份或动机。
6. 先区分研究条件与可交易性；受限或未知时按决策完整性规则降级。
7. 生成条件化次日计划，不得直接给出买卖指令。

## 每只股票必须输出

- 状态：“观察”“条件满足后可介入”“不适合新开仓”或“数据不足”。英文枚举
  只写入 context JSON，不得显示在 HTML 正文中。
- 至多三项支持事实，并写明日期与来源。
- 至少一项反面事实或未解除风险。
- 精确的触发条件；可以是价格/量能/板块条件，但必须是条件化表述。
- 精确的失效或不介入条件。
- 当前可交易性、限制原因和次日复核条件。
- 预先确定的 T+1 同期评价基准；无法可靠确定时显示“无法评价”。
- 相关事件、流动性、板块和市场状态风险。
- 最多两条重要行为结论；每条包含事实、计算、有限解读、反证、次日观察、证据
  充分度和数据缺口。没有足够证据时明确显示“数据不足”。

任何用于次日结论的触发或不介入条件，必须同时写入 context 的原子条件对象。
“板块承接良好”等无法量化的内容只能作为解读或风险，不得作为可复盘条件。

不得仅凭公告标题推断其会产生长期财务影响。
